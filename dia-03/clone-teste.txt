# git clone https://github.com/gabriel-felix-dev/primeiro-repo-github.git

    - Faz o clone de um repositório remoto na máquina

# git log

    - Ver o histórico de commits do repositório

# git remote -v

    - Mostra a origin do repositório.

# git clone https://github.com/gabriel-felix-dev/primeiro-repo-github.git repo-clonado

    - Faz o clone de um repositório remoto na máquina e cria uma pasta com um nome especifico.

# Quando o usuário tem um repositório na máquina, ele trabalha apenas no local. Ele pode adicionar 
  os arquivos para a stage e realizar os commits para versionar os documentos, mas ficará só na máquina dele.
  Nesse caso, ele precisará iniciar o repositório local, criar um repositório remoto e depois realizar a conexão
  do repositório local com o remoto.

# Já quando o usuário clona um repositório já existente, ele não precisa iniciar um repositório local, pois um
  remoto já existe e já está configurado.

# Sequência de comandos utilizados:

## git branch feature/header - Cria uma branch feature/header;

## git branch feature/footer - Cria uma branch feature/footer;

## git branch feature/sidebar - Cria uma branch feature/sidebar;

## git branch - Informa as branch que estão configuradas;

    feature/cadastro
    feature/calculadora
    feature/footer
    feature/header
    feature/login
    feature/sidebar
    * main
    next
    sprint

## git checkout feature/header - Muda para a branch feature/header;

## touch header.txt - Cria o arquivo header.txt;

## git add header.txt - Adiciona o header.txt à stage;

## git commit -m"feature/header: criação da documentação do header" - Versiona o arquivo da stage;

## git push -u origin feature/header - Envia o PR para o repositório remoto;

## PR - GitHud:

### Title: feature/header: criação da documentação do header;

### Description: Criação da documentação do header;

## git checkout feature/sidebar - Muda para a branch feature/sidebar;

## touch sidebar.txt - Cria o arquivo header.txt;

## git add sidebar.txt - Adiciona o sidebar.txt à stage;

## git commit -m"feature/sidebar: criação da documentação do sidebar" - Versiona o arquivo da stage;

## git push -u origin feature/sidebar - Envia o PR para o repositório remoto;

## PR - GitHud:

### Title: feature/sidebar: criação da documentação do sidebar;

## git checkout feature/footer - Muda para a branch feature/footer;

## touch footer.txt  - Cria o arquivo sidebar.txt;

## git add footer.txt - Adiciona o footer.txt à stage;

## git commit -m"feature/footer: criação da documentação do footer" - Versiona o arquivo da stage;

## git push -u origin feature/footer - Envia o PR para o repositório remoto;

## git merge feature/footer - Faz merge da branch feature/footer na branch sprint;
